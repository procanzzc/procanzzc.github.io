Array

  • in_array适用于索引数组.
  • in_array(‘参数’,’数组’);
  • in_array($userName,$arr);

HTTP协议

  • http协议是由客户机到服务器的请求和从服务器到客户机的响应进行了约束和规范
  • 请求由客户端发起,其规范格式为:请求行.请求头.请求主体.
  • 响应由服务器发出,其规范格式为:状态行.响应头.响应主体.
  • 200表示响应成功,304文档没有修改(缓存),403没有权限,404没有找到资源.
  • XMLHttpRequest 浏览器内置对象,用于后台与服务器交换数据.

POST

GET

GET与POST

  • GET不需要设置请求头,请求体send不需要设置参数
  • GET的数据传递通过url进行传递,而POST在send方法中传递
  • GET大小限制为4k.POST对大小没有限制

XML

  • 1.是一种标记语言,用来传输数据.
  • 2.语法必须要有一个根元素.标签可以有字母下划线数字组成,但是不能有空格.否则会传输为空.
  • 3.解析过于庞大而且冗余.
  • 4.读取xml文件,php>$xml = file_get_contents(‘demo.xml’);
  • 5.前台接收xml数据,var result = xhr.responseXML;

JSON

  • 1.javascript object notation 轻量级文本数据交换格式
  • 2.以键值对的形式存在,最后一个键值对不需要带逗号,不能在json文件中添加注释,否则解析错误.
  • 3.{}保存对象.[]保存数组.
  • 4.javascript解析方法: JSON.parse()从字符串解析出json对象
  • 5.javascript解析方法: JSON.stringify() 从json对象中解析出字符串
  • 6.读取数据格式文件 file_get_contents() 返回字符串,所以需要将返回的字符串转换为对象;

JSON.stringify()

  • stringify 的参数 第一个参数为转换的数据,第二个参数为转换的数据包含的内容,可以传入需要的keys数组,也可以传入回调函数来处理传输的数据.第三个参数为格式化字符串数据用的.

AJAX的封装

  • 1.获取请求头的content-type类型判断数据类型getRequestheader,如果是xml类型则使用responseXML的方式解析,如果是json类型则使用JSON.parse()方式进行解析,将字符串转换为json对象.
  • 2.获取get方法中url?后面拼接的字符串.
  • 3.get方法不需要设置请求头,请求行不需要设置参数,post方法需要.
  • 4.需要判断服务器返回的数据类型分别解析.
