沙箱与es6

沙箱模式

  • 模拟一个环境,js中的沙箱:函数
  • 沙箱的写法:所有的变量声明在沙箱的顶部,中间部分写功能代码,如果有需要,底部提供外部访问的接口

    jquery中的沙箱

jquery中的沙箱

  • 使用全局的window对象,代码的功能不会收到影响,但是这个window是一个全局的内置对象,后期无法做代码压缩
  • 沙箱模式的宗旨就是进行代码隔离,外部不能访问沙箱内部,沙箱内部也不直接使用外部的变量,所以不实用window,而使用jquery

forEach

  • 遍历数组

map

  • map会将结果存储到一个新生成的数组中再返回

click

  • js的onclick只能绑定一个单击事件,如果重复添加,后面的事件会覆盖前面的事件

函数调用模式

  • 函数调用模式 直接调用函数 .this指向window —>自调用模式
  • 方法调用模式 对象.方法() this执行调用当前函数的对象
  • 构造函数调用模式 new 函数() this指向通过当前构造函数创建出的对象
  • 上下文调用模式，传入的应是一个引用类型,如果传入的值类型的数据,默认会获取到这些数据的引用类型
  • 如果传入null或者undefined.默认得到window
  • 在调用构造函数的时候,不要使用call/apply来改this的指向,否则报错,new会将后面的归集为一体.报错：构造函数.call is not a constructor

    call(this指向的对象,参数,参数,参数…)
    apply(this指向的对象,[对象])

  • apply可以将伪数组中的数据拆分为一个一个的值,分别传给形参 – 自动拆解

promise

  • promise是一个构造函数,其中的第一个参数是一个函数,并且传入2个参数resolve,reject,分别表示异步操作执行成功之后的回调函数和异步操作执行失败之后的回调函数
  • resolve是将Promise的状态置为fullfiled，reject是将Promise的状态置为rejected
  • new Promise后会自动执行,所以我们一般使用的时候是将这个函数包含在一个函数中,在需要的时候去执行这个函数
  • promise可以解决多层回调时的麻烦,可以在then中返回promise对象然后继续调用then进行回调操作
  • catch方法会捕获rejected中的reason，如果在then函数中发生了错误，代码会继续运行下去，但报错的原因会归集到reason中输出
  • all方法会将then中返回的数据全部归集到一个数组里存储在result参数中

上下文调用模式的使用

  • argument是系统自带的,不可以修改删除,只能添加
  • 借用构造函数继承,可以继承在构造函数中this声明的成员,无法获取在构造函数原型中的成员
