Learn angular

发表于 2017-02-06 | 分类于 js
angular初步认知

angular 初步

  • 每个指令属性都有优先级
  • 内置模块系统
    • 这里的模块系统与requireJS模块类库有很大差别，它不是模块加载器
    • 而是帮我们组织代码结构的一套规范，严格限制了我们的编码方式
    • 同时多个模块可以通过依赖配置实现功能合并与共享
  • 内置模版引擎
    • 这里的模版有我们熟悉的表达式和过滤器系统，写法基本类似
    • 不同的是模版与数据的绑定方式不一样
  • 扩展原生HTML功能
    • 通过自定义属性与标签扩展了HTML功能，这些属性标签称为指令
    • 有了自定义指令基本上我们不再需要手动操作DOM，因为封装好了
  • 数据绑定
    • 一般的模版引擎，数据与视图的联系是一次性的，模版渲染到页面中俩者再无瓜葛
    • Angualr中数据与视图的联系是永久性的，一旦有了关联，任何时间任意一方改变都会引起另一方的变化

      angular模块

  • 主要是为了划分大型项目中不同的程序功能
  • 主模块的另一个作用:关联视图

angular控制器

  • 主要是实现页面数据动态化
  • 使用控制器需要配合使用$scope服务,这是视图与控制器的一个桥梁,给这个对象添加的属性都可以在视图中访问
  • 一个主模块可以创建任意多的控制器,每个控制器可以关联视图中的一个部分

angular服务

  • 主要是抽取公共逻辑或者分担控制器的压力
  • 服务创建的方式比较多,但是大同小异
  • 服务里面需要返回一个暴漏给外部的对象,方便外部调用
  • 在控制器调用的时候要引入服务依赖,并在回调函数中引入该服务的对应参数

内置服务

$http

  • $http提供了get/post方法,通过then方法来添加成功与失败的回调
  • post方法需要手动将参数修改为str字符串,将headers改为formdata的请求头

$location

  • 对浏览器location对象的封装
  • 这里$location参照的是前端路由来获取的path等值

$timeout与$interval

  • 原生的定时器改变数据后，angular是无法检测到的，造成视图不刷新
  • 这两方法是对原生方法的封装，使用方式一样，就是为了解决上诉问题的

过滤器

  • 主要是抽取模版中的公共数据处理逻辑
  • 相比服务更加的直观简洁
  • 过滤器要求callback再返回一个callback,外层callback接收依赖服务,内层callback是接收来自模版的值,做好加工处理返回
  • 封装视图中重复的数据操作,最常见的场景是数据格式的转换
  • 在js中可以使用$filter服务获取到指定的过滤器关键函数.然后在js中调用

ng-指令

  • ng-repeat
  • ng-model

  • 与表单实现双向数据绑定

  • ng-cloak

  • ng-if
  • 控制元素的插入与移除
  • ng-show/ng-hide
  • 控制元素的显示与隐藏
  • ng-repeat
  • 可以遍历数组或者对象,可以指定遍历次数 limitTo : num

内置过滤器

常用内置过滤器

  • currency可以将数值格式化为货币,可以指定货币符号与保留的小数点位数,默认保留2位
  • date 日期格式化
  • orderBy 对数组进行排序,一般与ng-repeat配合使用,第一个参数可以指定排序规则，值可以是字符串、数组,第二个参数可以指定顺序，默认false从小到大，true从大到小,了解：第一个参数还可以是数组,按优先级指定多个关键字或函数动态设置关键字
  • fliter 搜索过滤器,设置一个参数指定要查找的关键字,可以是字符串.对象
